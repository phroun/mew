Ok, now we are going to add support for color Schemes.
Every trinket should have a Scheme set, which is an integer, and defaults to -1.
If it is set to -1, it inherits the Scheme of its container.
All windows inherit their scheme from their MDI parent.
All popups inherit their scheme from the container that spawned them.
The menubar, statusbar, and dock inherit their scheme from the Desktop.
All regular windows on the desktop inherit their scheme from the Desktop itself.
The Desktop's scheme defaults to 0.
We will pre-initialize scheme 0 and set it up with default values.
If an invalid/undefined scheme number (other than -1) is ever specified for any trinket, it shall use scheme 0 instead.

Each of the colors in a scheme can be an entire ANSI sequence, or a nil/blank value.
When it is nil, it has a special meaning, which is given in the list below.

Any time i use the word "inherit" or "inherited" in the color descriptions below, it refers to the current behavior of inheriting the background color from the container, which will still be a concept in the new system.  Note that this has nothing to do with the Scheme inheritance spoken of above.
